Description

g_element is a container element for grouping together related graphics elements).
See https://www.w3.org/TR/SVG/struct.html#Groups 5.2.1 Overview. Container element is an element which can have graphics elements and other container elements as child elements. Container elements are: a, clipPath, defs, g, marker, mask, pattern, svg, switch, symbol, and unknown.

A group of elements, as well as individual objects, can be given a name using the id attribute. All g_elements should include an id as a string like id="background", or a macro defined in ../include/boost/svg_plot/detail/svg_style_detail.hpp to index std::string document_ids_

For example, an image background rectangle with border and fill: 

g0.svg_style_.fill_color(red); // Set svg_style color.

<g id="background" fill="rgb(255,255,255)"> <rect width="500" height="350"/> </g>

  33. virtual void write(std::ostream & os);
    Write function outputs appropriate SVG commands for a leaf or child element.

    Output all children groups (leafs of tree) of a group element. Example: <g fill="rgb(255,255,255)" id="background"><rect x="0" y="0" width="500" height="350"/></g>

    Do not output anything if no child leafs to avoid useless output. Example: <g id="legendBackground"> </g> 

  34. void write_attributes(std::ostream & os);

    Classes inherited from svg_element add other references, see section 5.3.1, like color, fill, stroke, gradients...
